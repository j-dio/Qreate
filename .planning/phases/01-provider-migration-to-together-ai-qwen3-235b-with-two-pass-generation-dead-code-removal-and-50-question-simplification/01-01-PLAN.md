**Define a Zod schema to validate Pass 1 JSON output:**
```typescript
const ConceptAssignmentSchema = z.object({
  id: z.number(),
  concept: z.string().min(3),
  type: z.enum(['multipleChoice', 'trueFalse', 'fillInTheBlanks', 'shortAnswer']),
  difficulty: z.enum(['easy', 'moderate', 'hard']),
  bloomLevel: z.enum(['remember', 'understand', 'apply', 'analyze', 'evaluate']),
  answerPosition: z.enum(['A', 'B', 'C', 'D']).optional(),
})

const TopicPlanSchema = z.object({
  topic: z.string().min(1),
  totalConcepts: z.number().min(1),
  concepts: z.array(ConceptAssignmentSchema).min(1),
})
```

**Implement `testConnection()` method:**

**Implement `generateExam(config, sourceText)` method as the two-pass orchestrator:**

1. Call `this.extractTopicPlan(config, sourceText)` -- Pass 1
2. Validate the plan with Zod schema
3. Call `this.generateFromPlan(plan, config, sourceText)` -- Pass 2
4. Return the formatted exam text

If primary client fails on either pass, retry with fallback client (if available). Max 2 retries per pass per client.

**Implement Pass 1: `private async extractTopicPlan(config, sourceText)`:**

Build a prompt that asks the model to analyze the source material and output a JSON plan. Key prompt elements:
- System message: "You are an expert educational assessment planner. Output valid JSON only."
- User message contains:
  - The source material (truncated to 80,000 chars max)
  - Total questions needed: `config.totalQuestions` (max 50)
  - Question type distribution from `config.questionTypes`
  - Instructions: "Extract N unique concepts from this material. Each concept must be completely distinct -- no overlap. Assign each concept a question type, difficulty level, and Bloom's taxonomy level."
  - For MCQ concepts, pre-assign answer positions (cycle A, B, C, D) to prevent answer bias
  - Explicit negative constraints: "Do NOT repeat any concept. Do NOT assign the same topic to multiple questions."
  - Output format: the TopicPlan JSON schema as an example

API call parameters:
```typescript
{
  model: this.primaryModel,  // or fallback model
  messages: [{ role: 'system', content: systemPrompt }, { role: 'user', content: userPrompt }],
  response_format: { type: 'json_object' },
  temperature: 0.3,
  max_tokens: 2048,
  top_p: 0.85,
}
```

Parse the response as JSON. Validate with `TopicPlanSchema.parse()`. If validation fails, throw with details.

**Implement Pass 2: `private async generateFromPlan(plan, config, sourceText)`:**

Build a prompt that takes the validated topic plan and generates the actual exam questions. Key prompt elements:
- System message: "You are an expert exam question writer. Generate exactly one question per concept in the plan. Follow the assigned type, difficulty, and Bloom's level exactly."
- User message contains:
  - The topic plan JSON (from Pass 1)
  - The source material (truncated to 80,000 chars max)
  - Format examples for each question type (MCQ with 4 options, T/F, Fill-in-blank with underscore, Short Answer)
  - Explicit constraints: "Base every question ONLY on the source material. Do NOT use 'All of the above'. Do NOT repeat any concept. For MCQ questions, place the correct answer at the position assigned in the plan."
  - Output format specification with `General Topic:`, numbered questions grouped by type, `----Answer Key----` section

API call parameters:
```typescript
{
  model: this.primaryModel,  // or fallback model
  messages: [{ role: 'system', content: systemPrompt }, { role: 'user', content: userPrompt }],
  temperature: 0.5,
  max_tokens: 16384,
  top_p: 0.9,
}
```

No `response_format` for Pass 2 (output is formatted text, not JSON).

**Implement `private async callWithFallback(fn)` helper:**
Wraps any API call function. Tries primary client first (2 retries, 2s exponential backoff). If all retries fail, tries fallback client (2 retries). Throws if both fail.
